In neuroimaging the typical analysis approach is a “mass univariate”, where a univariate model is independently fit to each voxel (volume element) in parallel. This approach can be applied to a large range of datasets including: functional MRI, anatomical MRI (with Voxel-Based Morphometry), PET, EEG and MEG data.

The three major software analysis package for neuroimaging: SPM (Statistical Parametric Mapping) and FSL (FMRIB Software Library) and AFNI (AFNI) provide implementations of mass univariate analyses. While there are many commonalities across software packages, there are also software-specific outputs that can be of interest for the end-user.

This document describes the encoding of results of a mass univariate neuroimaging analysis using the NIDM data model ([[!KEATORNI13]]). The goal of this specification is to provide a unified representation of neuroimaging results across analysis software. When a piece of information is only available in a specific software, software-specific extensions are provided.

Introduction

NIDM-Results is a NIDM compliant structured representation of the results of a massively univariate neuroimaging study.

NIDM-Results specification formally describes the encoding of the massively univariate neuroimaging results into a NIDM representation. Extensions are provided to describe software-specific objects for SPM (Statistical Parametric Mapping) and FSL (FMRIB Software Library) analysis software.

Structure of this Document

Section 2 provides an overview of NIDM-Results, distinguishing a core set of types and relations, commonly found in provenance, from extended structures catering for more specific uses. It also introduces a modular organization of the data model in components.

Section 3 illustrates how NIDM-Results can be used to express the results of a neuroimaging study in SPM and FSL.

Section 4 provides the definitions (and examples) of NIDM-Results concepts, structured according to three components.

Overview

This section introduces neuroimaging results concepts with informal explanations and illustrative examples (e.g. see SPM results). NIDM-Results distinguishes software-agnostic structures, forming the essence of the results, from software-specific structures catering for more specific uses of results by different analysis software. Core and extended structures are respectively presented in Section 2.1 and Section 2.2.

Domain covered by NIDM-Results

NIDM-Results is concerned with the modelling of model fitting and inference in the context of massively univariate analyses. A typical example is the analysis of functional MRI data but studies involving other modalities (such as PET) and sequences (e.g. anatomical MRI through VBM) can also be modelled. Data

The Data entity used by the Model Parameters Estimation entity represents the data used in model fitting. The optional attribute nidm:grandMeanScaling of type xsd:boolean indicating the presence or absence of grand mean data scaling. If nidm:grandMeanScaling = "true" then the target intensity can be indicated using the attribute nidm:targetIntensity with type xsd:float. Due to inaccuracies in the mask used to compute the mean intensity prior to scaling, the in-brain mean intensity in the scaled data may not equal the nidm:targetIntensity exactly. The median intensity of the Data can be used as a diagnostic to see if the typical brain intensity has been scaled to approximately the targetIntensity this can be encoded using the attribute nidm:medianIntensity of type xsd:float. See Data for an example in PROV-N notation.

Design matrix

The Design matrix entity used by the Model Parameters Estimation entity represents the design matrix used in model fitting. The optional attribute prov:location specifies a deferenceable URL to a file encoding the design matrix. Alternatively the relative (or absolute) path to a file could be used. The optional attribute nidm:visualization can be used to link to a graphics file representation of the design matrix (e.g. similar to the design matrix image in Pthe SPM software package). Again this path would ideally be a dereferencable URL but could alternatively be a relative or absolute path to the graphics file. See Design Matrix for an example in PROV-N notation.

Parameter Estimate Map

The Parameter Estimate Map entity is used to represent the calculated parameter array (e.g. "betamap" in a typical voxelwise GLM analysis). The entity is associated with the Model Parameters Estimation activity and used by the Contrast Estimation activity. The optional attribute prov:location specifies a deferenceable URL to a file representing the parameter array. Alternatively the relative (or absolute) path to a file could be used. The optional attribute nidm:originalFileName is used to retain the filename from the original software package that created it in case the prov:location was set to a URL. This feature was added to account for the way, for example, the SPM software names the parameter array images "beta_XXXX.img" and its correspondence to particular model parameters. The optional attribute nidm:coordinateSpace is used to associate spatial metadata with a physical file (see Coordinate). The optional attribute crypto:sha512 provides a unique sha-512 hash of the file pointed to by the prov:location attribute. See ParameterEstimateMap for an example in PROV-N notation.

Residual Mean Squares Map

The Residual Mean Squares Map entity is used to represent the differences between an observed value and the estimated function value where the function estimates are modeled using the ParameterEstimateMap entity. The entity is associated with the Model Parameters Estimation activity and used by the Contrast Estimation activity. The optional attribute prov:location specifies a deferenceable URL to a file representing the residuals array (e.g. in the case of voxelwise fMRI analyses this would be an image with voxel values equal to the residuals). Alternatively the relative (or absolute) path to a file could be used. The optional attribute nidm:originalFileName is used to retain the filename from the original software package that created it in case the prov:location was set to a URL. The optional attribute nidm:coordinateSpace is used to associate spatial metadata with a physical file (see Coordinate). The optional attribute crypto:sha512 provides a unique sha-512 hash of the file pointed to by the prov:location attribute. See ResidualMeanSquaresMap for an example in PROV-N notation.

Contrast Weights

The Contrast Weights entity is used to represent the linear combinations of parameter estimates with respect to the Design Matrix and used in creating the ContrastMap array. The attribute prov:value specifies the contrast vector. The attribute prov:type identifies the type of contrast weight. For example, prov:type=nidm:TContrastWeight identifies the contrast vector as a contrast on a t-statistic. The optional attribute prov:label provides a location for a human-readable, descriptive label for the contrast vector. The optional attribute nidm:contrastName corresponds to the contrast vector name in the generating software (i.e. SPM, FSL, AFNI, etc..). See ContrastWeights for an example in PROV-N notation.

Mask Map

The Mask Map entity is used to represent a map of the array values used in computing a ContrastMap array. In the case of voxelwise fMRI analyses this is typically an in-brain mask. The entity is derived from the Model Parameters Estimation activity and used by the Contrast Estimation activity. The optional attribute prov:location specifies a deferenceable URL to a file representing the mask array. Alternatively the relative (or absolute) path to a file could be used. The optional attribute nidm:originalFileName is used to retain the filename from the original software package that created it in case the prov:location was set to a URL. The optional attribute nidm:coordinateSpace is used to associate spatial metadata with a physical file (see Coordinate). The optional attribute crypto:sha512 provides a unique sha-512 hash of the file pointed to by the prov:location attribute. See MaskMap for an example in PROV-N notation.

Contrast Map

The Contrast Map entity is used to represent a map of selected parameters from the ParametersEstimateMap consistent with a set of ContrastWeights. The attribute prov:location specifies a deferenceable URL to a file representing the selected parameter estimates (e.g. in the case of voxelwise fMRI analyses this would be an image with voxel values equal to the parameter estimates). Alternatively the relative (or absolute) path to a file could be used. The attribute prov:type identifies the map as of type nidm:ContrastMap giving semantic meaning to the map. The optional attribute nidm:originalFileName is used to retain the filename from the original software package that created it in case the prov:location was set to a URL. The optional attribute nidm:coordinateSpace is used to associate spatial metadata with a physical file (see Coordinate). The optional attribute prov:label provides a human readable description of the contrast map. The optional attribute crypto:sha512 provides a unique sha-512 hash of the file pointed to by the prov:location attribute. See ContrastMap for an example in PROV-N notation.

Contrast Standard Error Map

The Contrast Standard Error Map entity is used to represent the standard errors for the contrast estimates represented in the Contrast Map entity and generated by the Contrast Estimation activity. The attribute prov:location specifies a deferenceable URL to a file representing the contrast standard errors array (e.g. in the case of voxelwise fMRI analyses this would be an image with voxel values equal to the standard errors). Alternatively the relative (or absolute) path to a file could be used. The optional attribute nidm:originalFileName is used to retain the filename from the original software package that created it in case the prov:location was set to a URL. The optional attribute nidm:coordinateSpace is used to associate spatial metadata with a physical file (see Coordinate). The optional attribute prov:label provides a human readable description of the contrast standard error map. The optional attribute crypto:sha512 provides a unique sha-512 hash of the file pointed to by the prov:location attribute. See ContrastStandardErrorMap for an example in PROV-N notation.

Statistic Map

The Statistic Map entity is used to represent a map of statistics derived from selected parameters from the ParametersEstimateMap consistent with a set of ContrastWeights. The attribute prov:type identifies the type of statistic in the map (e.g. nidm:TStatisticMap) giving semantic meaning to the map.The attribute prov:location specifies a deferenceable URL to a file representing the Statistic Map (e.g. in the case of voxelwise fMRI analyses this would be an image with voxel values equal to the statistic). Alternatively the relative (or absolute) path to a file could be used. The optional attribute nidm:originalFileName is used to retain the filename from the original software package that created it in case the prov:location was set to a URL. The optional attribute nidm:coordinateSpace is used to associate spatial metadata with a physical file (see Coordinate). The optional attribute prov:label provides a human readable description of the Statistic Map. The optional attribute crypto:sha512 provides a unique sha-512 hash of the file pointed to by the prov:location attribute. See StatisticMap for an example in PROV-N notation.

Model Parameters Estimation

The Model Parameters Estimation activity represents the computation of model parameters. The activity uses the DesignMatrix, Data, and MaskMap entities. The activity generates the ParameterEstimateMap, ResidualMeanSquaresMap, MaskMap, and the ReselsPerVoxelMap entities (see figure ). The attribute prov:label provides a human readable description of the activity. The attribute prov:type identifies the map as of type nidm:ModelParametersEstimate giving semantic meaning to the activity. See ModelParametersEstimation for an example in PROV-N notation.

Contrast Estimation

The Contrast Estimation activity represents the computation of the ContrastMap from the ParametersEstimateMap using a set of ContrastWeights. The activity uses the DesignMatrix, ParametersEstimateMap, ResidualMeanSquaresMap, and MaskMap entities. The activity generates the StatisticMap, , ContrastStandardErrorMap, and ContrastStandardErrorMap entities (see figure ). The attribute prov:label provides a human readable description of the activity. The attribute prov:type identifies the map as of type nidm:ContrastEstimation giving semantic meaning to the activity. See ContrastEstimation for an example in PROV-N notation.

Coordinate

The Coordinate entity encodes a point in N-dimensional space where the dimensionsInVoxels could be arbitrary. Principally the entity is used to specify the prov:location of a Peak. The attribute prov:type specifies the entity type as a both a prov:location and a nidm:Coordinate. The attributes nidm:coordinate[K] specifies the coordinate along the Kth dimension in voxels.

Coordinate Space

The Coordinate Space is an entity with spatial attributes (e.g. dimensionsInVoxels, units, and voxel-to-world mapping) that provides context to a SpatialImage (e.g. a StatisticMap. The attribute prov:type specifies the entity type as a nidm:Coordinate. The attributes prov:label provides a human readable description of the coordinate space. The attribute nidm:voxelToWorldMapping specifies the affine transformation matrix of the mapping from voxel to world space. The attribute nidm:worldCoordinateSystem specifies the coordinate system (e.g. nidm:mniCoordinateSystem). The attribute nidm:voxelUnits specifies the units in each of the dimensionsInVoxels. The attribute nidm:voxelSize specifies the size of the voxels in nidm:voxelUnits. The attribute nidm:dimensionsInVoxels specifies the dimensionsInVoxels of the coordinate space. The attribute nidm:numberOfDimensions specifies the number of dimensionsInVoxels in the coordinate space.

HeightThreshold

The Height Threshold entity represents a numerical value that establishes a bound on a range of voxel-wise or vertex-wise defined statistics. The attribute prov:type specifies the entity type as a nidm:HeightThreshold. The attribute prov:label provides a human readable description of the threshold. The optional attribute nidm:pValueUncorrected specifies the uncorrected p-value associated with the prov:value threshold. The prov:value threshold is in units consistent with the StatisticMap used in the Inference activity. The optional attribute nidm:userSpecifiedThresholdType stores a string identifying the threshold type selected in the SoftwareAgent. The optional attribute nidm:pValueFWER specifies the corrected p-value (e.g. family-wise error rate;FWER). See HeightThreshold for an example in PROV-N notation.

ExtentThreshold

The Extent Threshold entity represents a numerical value that establishes a bound on a range of cluster-sizes (i.e. minimum cluster size) used when thresholding a statistic. The attribute prov:type specifies the entity type as a nidm:ExtentThreshold. The attribute prov:label provides a human readable description of the threshold. The optional attribute nidm:pValueUncorrected specifies the uncorrected p-value associated with the spm:clusterSizeInResels. The attributes spm:clusterSizeInResels and spm:clusterSizeInVoxels specify the minimum cluster size in voxels and resels respectively. The optional attribute nidm:pValueFWER specifies a corrected p-value associated with the spm:clusterSizeInVoxels (e.g. family-wise error rate;FWER). See ExtentThreshold for an example in PROV-N notation.

Inference

The Inference activity in this context represents the process of inferring the ExcursionSet from a StatisticMap. The activity uses the HeightThreshold, ExtentThreshold, StatisticMap, ReselsPerVoxelMap, and MaskMap. The inference activity generates the ExcursionSet and the additional metadata SearchSpace and StatisticalImageProperties. See Inference for an example in PROV-N notation.

ExcursionSet

The Excursion Set entity represents a set of map elements surviving a thresholding procedure and is generated by the Inference activity. The attribute prov:type specifies the entity type as a nidm:ExcursionSet. The attribute prov:label provides a human readable description of the excursion set. The attribute prov:location specifies a deferenceable URL to a file representing the excursion set (e.g. in the case of voxelwise fMRI analyses this would be an image with voxel values equal to the statistic). Alternatively the relative (or absolute) path to a file could be used. The optional attribute nidm:originalFileName is used to retain the filename from the original software package that created it in case the prov:location was set to a URL. The optional attribute nidm:coordinateSpace is used to associate spatial metadata with a physical file (see Coordinate). The optional attribute spm:maximumIntensityProjection The optional attribute specifies a deferenceable URL to a file representing the maximum intensity projection image of the excursion set. The optional attribute crypto:sha512 provides a unique sha-512 hash of the file pointed to by the prov:location attribute. See ExcursionSet for an example in PROV-N notation.

ClusterStatistic

The Cluster Statistic entity represents a statistic defined at the cluster-level in an excursion set. In SPM it is defined as the cluster size. The attribute prov:type specifies the entity type as a nidm:ClusterStatistic. The attribute prov:label provides a human readable description of the cluster statistic entity. The optional attribute nidm:qvalueFDR represents the p-value adjusted for multiple testing, controlling for the false discovery rate. The optional attribute nidm:clusterLabelId specifies the cluster number (e.g. the SPM software numbers the clusters). The optional attribute spm:clusterSizeInResels specifies the cluster size in resels. The optional attribute nidm:pValueFWER specifies the p-value adjusted for multiple comparisons, controlling for the family-wise error rate. The optional attribute nidm:pValueUncorrected represents the uncorrected p-value. The optional attribute nidm:clusterSizeInVoxels specifies the number of voxels contained in the cluster. See ClusterStatistic for an example in PROV-N notation.

Peak

The Peak Statistic entity represents a statistic defined at the peak-level in an excursion set. The attribute prov:type specifies the entity type as a nidm:Peak. The attribute prov:label provides a human readable description of the peak statistic entity. The attribute prov:location references a Coordinate entity using the an information resource identifier of the form niiri:[prov:label] where the prov:label is the label of the coordinate entity consistent with this peak statistic.The optional attribute nidm:qvalueFDR represents the p-value adjusted for multiple testing, controlling for the false discovery rate. The optional attribute nidm:pValueFWER specifies the p-value adjusted for multiple comparisons, controlling for the family-wise error rate. The optional attribute nidm:pValueUncorrected represents the uncorrected p-value. The optional attribute nidm:equivalentZStatistic specifies the z-statistic. See Peak for an example in PROV-N notation.

NIDM-Results Types and Relations.

Element 1: Model Fitting

ModelParametersEstimation

A ModelParametersEstimation ◊is a the process of estimating the parameters of a general linear model from the available data (sub-type of prov:Activity).

A nidm:ModelParametersEstimation ◊ has attributes:
  • prov:label: an OPTIONAL label describing the ModelParametersEstimation.
activity(niiri:model_pe_id,
    [prov:type = 'nidm:ModelParametersEstimation',
    prov:label = "Model parameters estimation"])

Data

A Data ◊is a "A collection or single item of factual information, derived from measurement or research, from which conclusions may be drawn." (sub-type of prov:Entity).

A nidm:Data ◊ has attributes:
  • prov:label: an OPTIONAL label describing the Data.
  • nidm:grandMeanScaling: an OPTIONAL binary flag defining whether the data was scaled. Specifically, "grand mean scaling" refers to multipliciation of every voxel in every scan by a common value. Grand mean scaling is essential for first-level fMRI, to transform the arbitrary MRI units, but is generally not used with second level analyses.
  • nidm:targetIntensity: an OPTIONAL value to which the grand mean of the Data was scaled (applies only if grandMeanScaling is true).
  • nidm:medianIntensity: an OPTIONAL value corresponding to the median intensity of the Data. When grandMeanScaling is TRUE, this is a useful diagnostic to see if the typical brain intensity has been scaled to approximately the targetIntensity.
entity(niiri:data_id,
    [prov:type = 'nidm:Data',
    prov:type = 'prov:Collection',
    prov:label = "Data" %% xsd:string,
    nidm:grandMeanScaling = "true" %%xsd:boolean,
    nidm:targetIntensity = "100" %% xsd:float,
    nidm:medianIntensity = "115.32" %% xsd:float])

DesignMatrix

A DesignMatrix ◊is a design matrix associated with a particular design (sub-type of prov:Entity).

A nidm:DesignMatrix ◊ has attributes:
  • prov:label: an OPTIONAL label describing the DesignMatrix.
  • prov:location:path to a csv file describing the design matrix: an OPTIONAL undefined.
  • nidm:visualisation: an OPTIONAL graphical representation of an entity.
  • nidm:originalFileName: an OPTIONAL original name associated with a file (without path) in the context of the analysis software package.
entity(niiri:design_matrix_id,
